Тестовое задание: Институт охраны здоровья
=========================================

Репозиторий с решением тестового задания (3 задачи) на позицию технического специалиста / системного аналитика.

Структура
---------

- `task1-web-schedule` — веб‑утилита для просмотра расписания и записей пациентов.
- `task2-python-analytics` — Python‑сервис для анализа загруженности врачей и неявок.
- `task3-apps-script-report` — скрипт Google Apps Script для ежедневного отчёта посещаемости.

---

## Task 1 — Веб‑утилита расписания (HTML + CSS + JS)

Простая веб‑страница для просмотра расписания и записей пациентов по врачам с фильтрами и агрегированной статистикой.

Функциональность
----------------

- Отображение расписания приёмов в таблице: дата, время, врач, специальность, кабинет, тип приёма, статус.
- Фильтрация по врачу, дате и статусу записи (записан, завершён, отменён, неявка).
- Динамическая перерисовка таблицы и блока статистики по отфильтрованным данным без перезагрузки страницы.
- Расчёт агрегатов по текущему фильтру: общее количество записей, число завершённых, отменённых визитов и неявок.

Техническая реализация
----------------------

- Полноценная веб‑страница: разделение на `index.html`, `style.css` и `script.js`.
- Данные о приёмах хранятся в массиве объектов в `script.js` (поля `appointmentId`, `doctorName`, `specialty`, `date`, `startTime`, `endTime`, `room`, `patientType`, `status`).
- Нетривиальная логика реализована на стороне клиента:
  - инициализация фильтров по уникальным значениям врачей и статусов;
  - применение фильтров к массиву записей;
  - перерасчёт агрегированной статистики;
  - генерация строк таблицы в DOM.
- Код структурирован: логика вынесена в функции (`initFilters`, `applyFilters`, `resetFilters`, `renderTable`, `updateSummary` и т.д.), нет «портянки» из одного обработчика.

## Task 2 — Python‑аналитика загруженности (скрипт + Docker)

Скрипт на Python, который читает данные о приёмах из JSON, агрегирует их по врачу и дню, рассчитывает показатели загруженности и сохраняет результат в CSV.

Функциональность
----------------

- Загрузка данных из файла `appointments.json` (эмуляция JSON‑ответа внешней медицинской системы).
- Предобработка:
  - приведение дат к `datetime`;
  - расчёт длительности приёма при наличии времени начала и окончания.
- Агрегация:
  - группировка по врачу, специальности и дате;
  - расчёт количества плановых, завершённых, отменённых визитов и неявок;
  - вычисление показателей `noShowRate` и процента загрузки (`utilizationPercent`).
- Сохранение итоговой таблицы в `doctor_day_stats.csv`.

Обработка ошибок и вывод
------------------------

- Основной пайплайн обёрнут в `main()` с конструкцией `try/except`:
  - при успешном выполнении выводится сообщение `Saved stats to ...` с полным путём к CSV;
  - при ошибке (нет файла, некорректный JSON, проблемы с данными) выводится человекочитаемое сообщение `Error during processing: ...` вместо полного traceback.[page:59]
- Вспомогательные функции проверяют наличие файла и корректность JSON, при необходимости выбрасывают осмысленные исключения (`FileNotFoundError`, `ValueError`).[page:59]

Запуск локально
----------------

```bash
cd task2-python-analytics
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

После запуска в текущей директории появится файл `doctor_day_stats.csv` с агрегированной статистикой.

Запуск в Docker
----------------

В корне репозитория:

```bash
docker build -t health-it-analytics .
docker run --rm -v "$(pwd)/task2-python-analytics:/app" health-it-analytics
```

- Образ содержит Python и необходимые зависимости (pandas).
- Результирующий `doctor_day_stats.csv` сохраняется в том же каталоге благодаря монтированию тома.

## Task 3 — Ежедневный отчёт в Google Sheets (Apps Script)

Скрипт Google Apps Script для Google Таблицы `Health IT Test – Daily Visits`, который по триггеру добавляет строки с ежедневной посещаемостью по врача

Функциональность
----------------

- Выполняет HTTP‑запрос к публичному JSON‑API с помощью `UrlFetchApp.fetch` и разбирает ответ через `JSON.parse` (демонстрация работы с внешним API и JSON‑данными).
- Использует внешний ответ как источник демо‑данных: для каждого элемента ответа API выбирается врач из локального массива `doctors` (справочник с полями `doctorName` и `specialty`), после чего формируется строка отчёта.
- Записывает в лист `DailyStats` следующие колонки:
  - `date` — дата формирования отчёта;
  - `doctor` — ФИО врача из массива `doctors`;
  - `specialty` — специальность врача;
  - `planned`, `completed`, `canceled` — числа плановых, завершённых и отменённых визитов;
  - `NoShowRate` — доля неявок;
  - `comment` — статус загрузки (`OK (...)` или `ERROR: ...`).

Обработка ошибок
----------------

- Основная логика `fetchDailyStats()` обёрнута в `try/catch`.
- При неуспешном HTTP‑коде (не 200) или исключении:
  - добавляется строка с текущей датой и пустыми полями врача/показателей;
  - в колонку `comment` записывается текст вида `ERROR: HTTP ...` или `ERROR: <сообщение>`.
- Это позволяет видеть в самой таблице, удачно ли отработало очередное обновление.

Интеграция с Google Таблицей
----------------------------

- Используется существующая таблица `Health IT Test – Daily Visits` и лист `DailyStats` с уже настроенными заголовками
- Функция `onOpen()` добавляет в интерфейс таблицы пользовательское меню:
  - «Отчёты → Обновить посещаемость за сегодня» для ручного запуска скрипта.
- Настроен time‑driven триггер (например, раз в час), который автоматически вызывает `fetchDailyStats()` и добавляет новые строки.

## Использование AI‑ассистентов

При выполнении задания я использовал AI‑ассистента (Perplexity / ChatGPT) как помощника для генерации черновых фрагментов кода и текстов, с последующей ручной доработкой.

- **Task 1 (web‑страница)**  
  AI помог с начальным каркасом HTML/JS для таблицы расписания и фильтров. Финальная структура функций, именование полей (`appointmentId`, `doctorName`, `startTime`/`endTime` и т.д.), логика фильтрации, подсчёта агрегатов и наполнение тестовыми данными были переработаны вручную под предметную область.

- **Task 2 (Python‑аналитика)**  
  AI использовался для подсказок по общей структуре пайплайна и Dockerfile. Предобработка данных, выбор метрик (noShowRate, utilizationPercent), логика группировки по врачу и дню, а также обработка ошибок в `main()` реализованы и отлажены вручную.

- **Task 3 (Apps Script + Google Sheets)**  
  AI помог с шаблоном обращения к `UrlFetchApp.fetch`, примером парсинга JSON и общей структурой обработки ошибок. Конкретная структура листа `DailyStats`, массив `doctors` с ФИО и специализациями врачей, маппинг внешних данных на строки отчёта и настройка триггеров/меню в таблице сделаны вручную.

AI‑генерация не использовалась для бездумной вставки кода: все фрагменты были проверены, адаптированы и упрощены под формат тестового задания.
